MtM+ Toolbox

    From ideation to launch, from coding to designing, I developed this application all on my own.

Project Details

Type

Company's mobile application (Previous full-time job)

Timeline

4 month (Mar-Jun 2018)

Team

Szu-Fang Wang

Tools

Sketch, Android Studio, GitHub

Contributions

Research, Wireframe, Visual Design, Coding
As an IoT startup, MtM+ (my previous working company) had developed multiple applications for several sensors. Sales usually confused which app they shoud use because some apps are similar with just one small different setting. With this inconvience, I started to have the idea to combine all applications into one.

Process Work


Step 1
Observation

By observing users while they are using, I understood which parts confused them. I also talked with them and asked them to share their previous experience separately. Trying to understand their pain points and expectations.

What problems have obsered?

  • Customers question sensors‘ performance and quality for the company, because sales chose the wrong app to demo.
  • Users cannot find the information or function they need sometimes. User flow should not be confused and should be as simple as possible.
  • Sales need to learn how to use the app before demo. Don't let sales learn how to use the application.

Step 2
Ideation

Based on what I have learned from observations and interviews, I started to list down ideas that might fix problems and become solutions. During the brainstorming, I constantly make sure that I was staying focused on the needs and desires of them rather than choosing the path that would be easier for me to develop.

What ideas have come up?

  • Simplifing and unifing the user flow of each application in Toolbox.
  • Connecting to product automatically and fixing all errors in each application.
  • Improving the design to make it more productive.


Step 3
Wirframing &
User Feedback

I sketched my ideas down into a wireframe. And invited company sales, which is the main user, to do the usability test by letting them see the wireframe and asked some questions (e.g. If you want to use a function where will you click? What do you expect after you click? ).

What design has been changed?

  • Increased the text size for main information so users don't need to "find" the information they need.
  • Provided more relative information such as distance, speed, and calories, which could be calculated based on the pedometer.

Step 4
Visual Design

After confirming the user flow, I started to focus on visual design from deciding the main colors, defining icons to designing each interface.

Step 5
Coding &
Launch

After designing is done, I started to code the application and to make all functions works. After a few times of testing the application is now can be downloaded on Google Play.
